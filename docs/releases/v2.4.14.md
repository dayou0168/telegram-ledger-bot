# Go v2.4.14

v2.4.14 completes broadcast upstream delivery context, media-aware reply flow, replacement combinations, final-only receipts, per-user message preferences, and fresh quick-reply authorization. It is a bot-side release with one forward-only PostgreSQL migration; the watcher protocol is unchanged.

## Broadcast and reply flow

- Upstream copies identify both the original sender and the effective chat, group, or all-groups target.
- Group replies preserve the original broadcast context and copy the real text, photo, and caption instead of substituting a summary.
- No transient “发送中” placeholder is sent. Operators receive only the final success or failure receipt.
- `chat`, `group`, and `all` selections use their respective target rules when resolving upstream recipients and receipts.

## Replacement and preferences

- A text-only single-chat broadcast can now be replaced by the configured fixed image.
- Text, photo, caption, fixed-text, and fixed-image combinations share the same replacement matrix across edit and fallback-send paths.
- Broadcast and reply receipt preferences are stored independently per observer/source pair.
- The observer grant is a ceiling: a personal preference cannot enable a channel that current observer authorization does not allow.
- Quick reply eligibility is the final reply-recipient set and is checked both before enqueue and immediately before send.

## Database and compatibility

- Adds migration `2.4.20-broadcast-message-preferences`, extending durable broadcast preferences for separate send and reply channels.
- The migration runs transactionally under the existing PostgreSQL advisory migration lock and is idempotent on fresh, upgraded, repeated, and concurrent startup.
- No watcher code, protocol, scheduler, database, or chain-source behavior changes.
- Production should update bot containers only. The release workflow still publishes matching watcher artifacts, but an existing v2.4.13 watcher does not need to restart.
