# Go v2.4.9

v2.4.9 hardens timeout ownership and fallback gap progress across the watcher and bot. It has no PostgreSQL migration or configuration change, but both components contain runtime fixes and must be upgraded together.

## Fixes

- Gives each Tronscan request one authoritative timeout cause. Parent scan-round cancellation does not cool healthy Keys, while a genuine independent transport timeout still does.
- Covers the equal parent/request deadline race across eleven Keys, including partial successful pages and wrapped late transport errors.
- Keeps fallback `window` tasks paging to the configured safety limit instead of treating an empty `end_page` as immediate completion.
- Makes ordinary gap Advance and Yield updates monotonic with `GREATEST(next_page, requested_page)`, so stale workers cannot rewind progress.
- Preserves explicit cursor reset only in the dedicated transactional Split path.

## Compatibility

- No PostgreSQL migration is added.
- No environment variable or configuration default changes.
- Existing readiness and fallback activation contracts remain unchanged.
- Watcher and bot must both run v2.4.9 because the fixes span Tron requests, fallback scheduling, and shared gap storage.

## Upgrade Order

1. Verify Release checksums and both immutable image digests.
2. Preserve runtime files and back up both PostgreSQL databases as an operational precaution.
3. Upgrade `ledger-chain-watcher` to v2.4.9 and require `/healthz`, authenticated `/status`, and `source_ready=true`.
4. Upgrade every bot instance to the v2.4.9 image.
5. Confirm fallback gap cursors never regress during ordinary Advance/Yield and window tasks continue to their safety limit.

Use [the v2.4.9 production rollout guide](../production-rollout-v2.4.9.md) for the exact acceptance boundary.

## Residual Acceptance

PostgreSQL integration, race, vet, build, eleven-Key deadline competition, and cursor regression gates cover this release. Real Tronscan timeout/throttling behavior, production fallback load, and real incoming/outgoing USDT transfer timing remain deployment acceptance items.
