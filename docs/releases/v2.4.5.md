# Go v2.4.5

v2.4.5 is a bot-focused broadcast administration release. It adds host-controlled transfer of broadcast-group management ownership, completes missing direct chat permissions atomically when explicitly requested, and restructures the group controls for desktop and mobile. The watcher code is unchanged.

## Broadcast group owner transfer

- Only the configured host can see or call the owner-transfer action.
- The target must be an active primary global operator.
- The transaction locks the group, verifies the submitted expected owner, and rejects stale or concurrent submissions without mutation.
- Existing broadcast permissions are preserved. Missing direct chat permissions are either reported exactly with no partial transfer, or inserted in the same transaction when the host explicitly enables completion.
- Auto-granted permissions record the host as grantor and append permission audit rows.
- Successful owner changes append immutable `broadcast_group_owner_transfer_events` and the existing group audit event.
- `created_by` remains historical evidence; the admin “创建者” column now displays the current manager represented by `owner_user_id`.
- Records the forward-only migration gate `2.4.5-broadcast-group-owner-transfer`.

## Admin interface

- Reorganizes create, rename, transfer, and delete controls into a compact responsive operation area.
- Keeps the selected group, target owner, and permission-completion choice after a `409 Conflict` response.
- Hides all owner-transfer controls from non-host administrators.
- Preserves primary-operator visibility and management isolation.

4. Verify host transfer with and without automatic permission completion, stale-owner conflict handling, and the non-host hidden/forbidden boundary.

Migrations are forward-only. Rolling the bot image back does not remove the transfer audit table, indexes, trigger, permission audits, or owner changes already committed. Existing ownership must not be bulk-rewritten during deployment. The release workflow republishes watcher assets for coherence, but production does not need to update or restart the watcher for v2.4.5.
